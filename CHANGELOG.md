#### v0.3.0

> 30 May 2026

**Parsing fidelity**
- feat: Docling structured-tree PDF parse — preserves page numbers, section hierarchy, and `type="table"` blocks (replaces markdown round-trip)
- feat: DoclingParser for `.docx`, `.pptx`, `.xlsx`, `.html` via same structured-tree path
- fix: `DocumentConverter` singleton — constructed once per process, not per file
- fix: tables written as `.md` (was `.csv`); `manifest.tables` now populated

**Chunking**
- fix: oversized blocks (> `max_tokens`) are now split with overlap; `page` and `section` metadata preserved on every sub-block

**Retrieval correctness**
- fix: FTS and vector indexes now store a content hash; stale indexes are automatically rebuilt on re-pack (no more silent stale results)
- feat: `agentpack index` command — pre-build both indexes explicitly

**Indexing reliability**
- fix: documents that fail to parse are no longer silently marked `success` with zero chunks — the manifest now records `status: failed` with the reason, and a warning is printed
- fix: failed parses are no longer written to the L1 cache, so a transient failure is retried on the next pack instead of being cached as empty forever

**Benchmarking**
- feat: comprehensive benchmark suite covering 9 retrieval strategies (Raw File, Naive Chunk, Naive Chunk + Vector, Recursive Chunk + Vector, Parent-Document, Cross-Encoder Rerank, AgentPack FTS / Vector / Hybrid) across two corpora — FinanceBench (homogeneous) and DocBench (heterogeneous)
- feat: both deterministic (Hit@3, MRR, Citation Precision) and generative (LLM-as-judge Correctness, Faithfulness, Answer/Context Relevance) results; see `BENCHMARK.md`

**Caching**
- feat: content-addressed cache (`cache.db`) with three layers: L1 parse, L3 embed, L5 query
- feat: L1 parse cache — unchanged files skip Docling entirely on re-pack
- feat: L3 embedding cache — unchanged chunks are never re-embedded

**Retrieval quality**
- feat: HNSW vector backend (`hnswlib`) as default; brute-force `np.dot` fallback retained
- feat: RRF (Reciprocal Rank Fusion) replaces min-max hybrid scoring
- feat: pre-normalized vectors at build time; query similarity is a plain dot product
- feat: embedding model singleton — loaded once per process
- feat: `--source`, `--section`, `--page` metadata filters on `agentpack retrieve`

**Performance**
- feat: parallel parsing via `ThreadPoolExecutor(max_workers=4)`; output order is stable

**CLI / config**
- feat: `agentpack.toml` config file (`[pack]` section); CLI flags override config
- feat: `--fast` flag unifies PyMuPDF + skip-HNSW mode; `--fast-pdf` kept as deprecated alias

**Tests**
- fix: repaired 5 pre-existing broken tests (`test_search_pack_hybrid`, `test_search_hybrid`, `test_retrieve_error_handling`, `test_run_eval`, `test_slice_financebench`)
- feat: 59 tests passing; new tests for cache, config, chunker oversize, end-to-end round-trip, RRF, invalidation, metadata filters
